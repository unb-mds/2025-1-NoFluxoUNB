1️⃣ Fluxo Geral — Caixa-Preta em 5 Passos

📐 FÓRMULA GERAL: aplicável a qualquer exercício de EP+AVL.
1
LER E IDENTIFICAR VARIÁVEIS Liste todas as variáveis de entrada. Para cada uma: qual o TIPO (faixa, conjunto, string, booleano, sub-regra)? Qual o DOMÍNIO? Tem sub-condição?
2
IDENTIFICAR CLASSES DE EQUIVALÊNCIA Para cada variável, defina classes válidas e inválidas usando os 7 tipos. Numere C1, C2, C3... agrupando por variável.
3
APLICAR AVL EM FRONTEIRAS NUMÉRICAS Para cada faixa numérica ou tamanho de string: ponto exato (válido) + vizinho (inválido). Calcule o step correto.
4
MONTAR TABELAS 1 E 2 Tabela 1 (#, Variável, Classe, Tipo, Repr.). Tabela 2 (Variável, Fronteira, Ponto exato, Vizinho).
5
PROJETAR CASOS DE TESTE (Tabela 3) Válidos = combinações da matriz. Limites = par exato+vizinho. Inválidos = um por classe, isolado.

2️⃣ Os 7 Tipos de Variável

Tipo Quando usar Exemplo do enunciado Classes AVL?
1
Faixa		 Variável numérica com mínimo E máximo peso entre 0,1 e 50 kg 3 (1V + 2I) SIM
2
Conjunto		 Lista FIXA de opções discretas listadas no enunciado tipo: NORMAL ou EXPRESSA N val. + 1 inv. NÃO
3
String		 Texto com formato/regex/tamanho obrigatório CEP: 8 dígitos 3+ (1V + 2+I) PARCIAL
4
Booleano		 Apenas dois estados (true/false) vip: sim ou não 2 (ambas V) NÃO
5
Igualdade		 Deve ser exatamente um valor específico código = 1234 3 (1V + 2I) SIM
6
Coleção		 Lista/array de itens (carrinho, lista de produtos) carrinho: 1+ itens 3 a 4 SIM
7
Sub-regra		 UMA classe válida com condição EXTRA que muda comportamento CEP válido MAS inicia 7/8/9 → +R$15 2+ válidas DEPENDE
⚠️ Conjunto (Tipo 2) × Sub-regra (Tipo 7): Conjunto = opções FIXAS listadas no enunciado ("X ou Y ou Z"). Sub-regra = UMA classe válida que se divide porque tem uma condição EXTRA dentro dela ("se for válido E inicia com 7/8/9").

🔑 Guia rápido de identificação

Tem mínimo E máximo numérico? ─────────→ TIPO 1 (Faixa)
Tem lista fixa de opções discretas? ───→ TIPO 2 (Conjunto)
Tem formato/regex/tamanho obrigatório? ─→ TIPO 3 (String)
É verdadeiro ou falso? ─────────────────→ TIPO 4 (Booleano)
Deve ser exatamente um valor X? ────────→ TIPO 5 (Igualdade)
É uma lista/array de itens? ────────────→ TIPO 6 (Coleção)
Comportamento DIFERENTE dentro da válida?→ TIPO 7 (Sub-regra)

  • Agrupar por variável
  • Válidas antes das inválidas
  • Representante NÃO na fronteira
  • Conjunto: usar o valor exato
  • Inválida de conjunto: valor concreto fora dele

4️⃣ Tabela 2 — Análise de Valor Limite (AVL)

Estrutura

Variável Fronteira Ponto exato Vizinho
peso mínimo 0,1 (válido) 0,05 ou 0,09 (inválido)
peso máximo 50 (válido) 50,1 (inválido)
valor (R$) mínimo 10,00 (válido) 9,99 (inválido)
nome (tamanho) mínimo 3 chars (válido) 2 chars (inválido)

Como calcular o vizinho (step)

Tipo da variávelStepExemplo
Inteiro1idade mín 18 → vizinho 17
Decimal R$ (2 casas)0,01R$10,00 → vizinho R$9,99
Decimal kg (1 casa)0,1 ou 0,010,1 kg → vizinho 0,0 ou 0,09
String (tamanho)1 caractere8 chars → vizinho 7 chars
⚠️ AVL NÃO se aplica em: conjuntos nominais (NORMAL/EXPRESSA), booleanos (true/false), strings sem regra de tamanho.
💡 Sub-regra (Tipo 7) tem fronteira própria: CEP válido inicia 7-9 → ponto exato "70000000" (com sub-regra), vizinho "60000000" (sem sub-regra). Os dois são VÁLIDOS, o que muda é o comportamento.

Coluna "Entradas"

  • Valor de retorno calculado (ex: R$ 30,00)

  • Nunca "erro" genérico

  • AÇÕES do usuário na UI
  • O que ele DIGITA, CLICA, SELECIONA
  • Sequência de interações

  • O que o usuário VÊ na tela

  • Campo VAZIO/NÃO PREENCHIDO (CEP, idade, etc)
  • Opção NÃO SELECIONADA (radio button vazio)

8️⃣ Caixa-Branca — MC/DC

MC/DC (Modified Condition/Decision Coverage): cada condição deve afetar INDEPENDENTEMENTE o resultado da decisão. Para N condições, geralmente N+1 testes.

📐 FÓRMULA GERAL — MC/DC em 6 passos

1
IDENTIFICAR DECISÕES Cada if, while, for é uma decisão. Liste todas como D1, D2, D3...
2
LISTAR CONDIÇÕES ATÔMICAS Cada sub-expressão booleana é uma condição. Ex: peso < 0.1 || peso > 50 tem C1=(peso<0.1) e C2=(peso>50).
3
DEFINIR T/F PARA CADA CONDIÇÃO Para cada condição: qual valor de entrada faz ela ser TRUE? E FALSE? Esta é a Tabela 1 do MC/DC.
4
MONTAR TABELA VERDADE Para decisões com 2+ condições. 2 condições = 4 linhas (TT, TF, FT, FF). 3 condições = 8 linhas.
5
IDENTIFICAR PARES DE INDEPENDÊNCIA Pergunta-chave: "Consigo mudar o resultado da decisão alterando APENAS UMA condição?" Para cada Ci: encontre 2 testes onde só Ci muda E o resultado muda.
6
MENOR CONJUNTO DE TESTES Selecione o menor conjunto que cobre todos os pares. Testes podem ser COMPARTILHADOS entre condições. Meta: N+1 testes para N condições.

Exemplo concreto — Decisão D1: peso < 0.1 || peso > 50

Linha C1 (peso<0.1) C2 (peso>50) D1 Par de independência
R1TFTC1: par (R1, R2)
R2FFFC1: par (R1, R2) ✓
C2: par (R3, R2) ✓
R3FTTC2: par (R3, R2)
R4TTT(redundante)

Pares de independência:

  • C1 afeta: R1 (T,F→T) vs R2 (F,F→F) — só C1 muda, D1 muda
  • C2 afeta: R3 (F,T→T) vs R2 (F,F→F) — só C2 muda, D1 muda

Conjunto mínimo: 3 testes (R1, R2, R3) cobrem 100% MC/DC desta decisão.

  • Operador AND: decisão T só quando TODAS são T. Use a linha "todas T" como pivô.
  • Para cada condição, varie ela mantendo outras fixas para mudar o resultado.

🔟 Armadilhas Comuns — O que a Professora Mais Cobra

🪤 1. Confundir Conjunto com Sub-regra

Conjunto: opções FIXAS listadas (NORMAL ou EXPRESSA).
Sub-regra: UMA classe válida com condição EXTRA (CEP válido MAS inicia 7-9).

🪤 2. Esquecer inválida do conjunto

Toda variável SEMPRE tem ao menos uma classe inválida. Inválida do conjunto = "qualquer valor fora das opções listadas".

🪤 3. Combinar inválidas

Inválido sempre ISOLADO. Outras entradas devem ser válidas. Senão a primeira exceção mascara a segunda.

🪤 4. Esquecer fronteiras de tamanho

Strings com tamanho mín/máx (senha 8-20 chars) TÊM AVL — sobre o tamanho.

🪤 5. Saída esperada genérica

Nunca "erro" sozinho. Sempre IllegalArgumentException("Peso inválido") ou valor calculado.

🪤 6. Representante na fronteira

Use o ponto interno (10, não 0,1). A fronteira é para a Tabela 2.

🪤 7. Esquecer classes do nível sistema

No sistema SEMPRE adicionar: carrinho vazio, campos não preenchidos, opções não selecionadas.

🪤 8. MC/DC — par de independência errado

Pares devem ter SÓ a condição alvo mudando E o resultado mudando. Se mais de uma condição muda, não é par válido.
