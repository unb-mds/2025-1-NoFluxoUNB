O usuário e sua equipe são **colaboradores externos** que vão **auditar e escrever testes** no projeto NoFluxo UnB — eles **não são os desenvolvedores do código de produção**.

Papel: auditores/testadores. Escrevem testes (unidade, integração, etc.) e produzem relatórios sobre o estado dos testes existentes.

Implicação prática para relatórios:
- Escrever sempre na perspectiva de avaliador externo ("a equipe analisou", "o projeto adota", "foi identificado").
- Não usar primeira pessoa coletiva referindo-se ao código de produção ("nosso código", "implementamos a feature X").
- Quando a equipe **escrever testes novos**, é correto usar primeira pessoa para os testes em si ("propusemos os testes a seguir", "reescrevemos o caso").
